A dev container file is a JSON configuration file, usually named devcontainer.json, stored in a project's .devcontainer folder. It defines how a development container should be created and configured for that project.

Use cases

The devcontainer.json file tells tooling (for example, VS Code Dev Containers or GitHub Codespaces) how to create a consistent development environment. Common options to specify include:

  • which container image or build definition to use
  • which editor extensions and settings to apply
  • how apps running in the container can be accessed from the host machine
  • which setup commands to run after container creation

Minimal Contents

At the top level, devcontainer.json is a JSON object. A practical minimum is an environment source, most commonly the image property. This is specified as an image reference (for example, mcr.microsoft.com/devcontainers/base:ubuntu).

The name property is recommended for readability but is not strictly required in every setup.

The following is an example of a simple devcontainer.json file, including the optional name property:

{
  "name": "My Development Container",
  "image": "mcr.microsoft.com/devcontainers/base:ubuntu"
}
